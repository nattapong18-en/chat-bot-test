## Documentation Rules

- Read the relevant documentation before modifying code.
- Do not build features that are outside the documented requirements.
- Do not change the technical stack without recording the decision in `15-decisions.md`.
- If the code and documentation conflict, stop and report the conflict.
- Update the task list after completing work.
