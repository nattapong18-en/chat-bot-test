# Design System

## Design Direction

The interface should feel:

- Calm.
- Minimal.
- Modern.
- Professional.
- Content-focused.
- Apple-inspired without copying Apple.
- Spacious.
- Free from unnecessary visual noise.

## Layout

- Estimated desktop sidebar width: 260–288 px.
- Estimated chat content width: 760–880 px.
- The composer should align with the message content width.
- Mobile navigation should use a drawer.

## Typography

Use a system font or Geist.

Main text roles:

- Page title.
- Section title.
- Body text.
- Secondary text.
- Code text.
- Metadata.

## Spacing

Use a consistent spacing scale:

- 4 px.
- 8 px.
- 12 px.
- 16 px.
- 24 px.
- 32 px.
- 48 px.

## Border Radius

- Small: 6 px.
- Medium: 10 px.
- Large: 16 px.
- Pill: 9999 px.

## Components

- Button.
- IconButton.
- Textarea.
- Tooltip.
- Sheet.
- Dialog.
- DropdownMenu.
- ScrollArea.
- Skeleton.
- Toast.
- MessageBubble.
- MarkdownRenderer.
- CodeBlock.
- PromptSuggestion.

## Accessibility

- Text contrast must be readable.
- Interactive elements must support keyboard input.
- Icon-only buttons must have accessible labels.
- Focus states must be visible.
- Color must not be the only way to communicate status.
